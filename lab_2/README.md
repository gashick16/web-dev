# Лабораторная работа №2
## Тема: Проектирование и реализация RESTful API

### Цель работы
1.  Изучить архитектурный стиль REST и принципы построения веб-сервисов.
2.  Освоить работу с HTTP-методами (GET, POST, PUT, PATCH, DELETE) и кодами состояний.
3.  Получить практические навыки подключения реляционной СУБД (PostgreSQL) к веб-приложению.
4.  Освоить механизм миграций базы данных для управления схемой данных.
5.  Изучить основы работы с ORM (Object-Relational Mapping) для абстракции работы с данными.
6.  Реализовать механизм мягкого удаления (Soft Delete) данных.
7.  Реализовать пагинацию для списковых методов API.
8.  Закрепить навыки контейнеризации приложений с помощью Docker и оркестрации сервисов через Docker Compose.
9.  Реализовать разделение ответственности (Separation of Concerns) между слоями приложения (Контроллер, Сервис, Модель/Репозиторий).

### Настройка
Создать вайл .env и заполнить все значения
```
DB_USER=
DB_PASSWORD=
DB_NAME=
DB_HOST=postgres
DB_PORT=
PORT=4200
NODE_ENV=development
```

### Запуск
```bash
docker-compose up
```

### Результат
Для проверки можно использовать Postman коллекцию, которая лежит в корне проекта

### Остановка
```bash
docker-compose stop
```
